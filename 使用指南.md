# workflow-kit 使用指南

> 版本: v2.1 · 2026-05-29

`workflow-kit` 是一套可迁移的大模型自动化开发工作流。它把需求输入、结构化确认、技术方案、落地计划、任务拆分、实现记录、测试验证、构建验收和全链路复核固定成可恢复的状态机。

workflow-kit 语言无关。Java/Maven/Jar 是默认构建验收 profile，不是项目硬依赖；非 Java 项目安装后覆盖 `docs/.workflow/project_config.json` 即可。

## 一、这是什么

你把 `workflow-kit` 安装到项目后，Codex 或 Claude Code 会按项目入口文件读取本地规范，不再靠临时口头约定工作。

它包含三层内容：

1. 工作流引擎：`docs/.workflow/`
2. 入口受管块模板：`docs/.workflow/templates/entry/`
3. 人类说明文档：`INSTALL.md`、`使用指南.md`

业务项目自己的需求文档、Bug 记录、知识库内容、根目录 OpenSpec 历史 change 不属于通用包。

## 二、包内文件

```text
workflow-kit/
├── AGENTS.md
├── CLAUDE.md
├── INSTALL.md
├── 使用指南.md
└── docs/
    ├── README.md
    ├── 01-features/
    ├── 02-bug-fix/
    ├── 03-knowledge/
    └── .workflow/
        ├── 工作流规范.md
        ├── project_config.json
        ├── scripts/
        │   └── install_entry.py
        └── templates/
            └── entry/
                ├── AGENTS.md.tpl
                └── CLAUDE.md.tpl
```

## 三、安装

```bash
mkdir -p your-project/docs/{01-features,02-bug-fix,03-knowledge}
cp -r workflow-kit/docs/.workflow your-project/docs/.workflow
cp -n workflow-kit/docs/README.md your-project/docs/README.md
cp workflow-kit/INSTALL.md your-project/
cp workflow-kit/使用指南.md your-project/
python3 your-project/docs/.workflow/scripts/install_entry.py --target your-project
```

目标项目已有 `AGENTS.md` 或 `CLAUDE.md` 时，安装脚本只新增或替换 `WORKFLOW-KIT` 受管块，标记外项目说明会保留。

## 四、安装验证

在目标项目根目录执行：

```bash
python3 docs/.workflow/scripts/init_feature.py --list
```

再确认入口文件包含受管块和关键路径：

```bash
grep -n "WORKFLOW-KIT:START" AGENTS.md CLAUDE.md
grep -n "docs/README.md" AGENTS.md CLAUDE.md
grep -n "工作流规范.md" AGENTS.md CLAUDE.md
grep -n "context_packets.py" AGENTS.md CLAUDE.md
```

构建产物模板在 `docs/.workflow/project_config.json`：

```json
{
  "build": {
    "artifact_pattern": "target/*.jar",
    "artifact_label": "Jar",
    "build_command": "mvn -DskipTests package",
    "build_record_keyword": "Jar"
  }
}
```

前端示例：

```json
{
  "build": {
    "artifact_pattern": "dist/**/*",
    "artifact_label": "frontend dist",
    "build_command": "npm run build",
    "build_record_keyword": "dist"
  }
}
```

Go 示例：

```json
{
  "build": {
    "artifact_pattern": "bin/*",
    "artifact_label": "Go binary",
    "build_command": "go build -o bin/app ./...",
    "build_record_keyword": "bin/"
  }
}
```

## 五、开始一个新需求

在 Codex 或 Claude Code 中直接说：

```text
我有一个新需求要做：……
```

主 Agent 会先确认输入模式：

- A：已有需求文档
- B：只有想法或口述描述

然后执行：

```bash
python3 docs/.workflow/scripts/init_feature.py "功能名"
```
