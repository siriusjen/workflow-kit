# F01 工作流自动化优化 技术方案 v1

> 日期: 2026-05-29
> 基线: `01-需求确认/需求说明书-v1.md`
> 执行原则: 先测试安全网，后行为重构；每个脚本改动在独立 git worktree 内完成。

## 总体设计

本 feature 采用分层渐进改造：

1. 先修复安装入口，使 workflow-kit 能可靠安装到目标项目且不覆盖用户入口文件。
2. 再补行为回归测试，锁定当前状态机、门禁和异常路径。
3. 随后把重复的阶段、门禁、子 Agent 和上下文定义抽到单一事实源。
4. 在单一事实源基础上，把 validator 绑定到 `stage_gates.py auto`，再将门禁前置条件声明式化。
5. 最后处理上下文包模板、公共工具、配置 schema 和低摩擦命令。

每个设计项都必须保持 S1-S10 阶段数量和门禁顺序不变；涉及 state 字段的新增必须向后兼容旧记录。

## D 编号方案

### D01 安装受管块与模板化入口

新增 `docs/.workflow/templates/entry/AGENTS.md.tpl` 和 `CLAUDE.md.tpl`，存放包含 `WORKFLOW-KIT` 标记的入口受管块。新增 `docs/.workflow/scripts/install_entry.py`：

- 从 `.workflow/templates/entry/` 读取模板，避免安装后依赖 workflow-kit 源根目录。
- 支持 `--target <项目根>`、`--only AGENTS.md|CLAUDE.md`、`--dry-run`。
- 目标文件不存在则创建，仅包含受管块。
- 目标文件存在且有完整受管块则替换整块。
- 目标文件存在但无受管块则在末尾追加，保留原内容。
- 目标文件只有 START 或 END 单侧标记时失败。

安装文档先创建 `your-project/docs/`，再复制 `.workflow` 和 `docs/README.md`，最后运行脚本。

覆盖: R01, R02

### D02 回归测试安全网

在 `docs/.workflow/scripts/tests/` 下扩展测试：

- `test_install_entry.py` 覆盖安装脚本。
- `test_workflow_gates.py` 追加 docs-only、worktree、artifact/http、人工锚点和 bug 链行为。

测试必须在生产代码改动前先失败，再实现通过。

覆盖: R03

### D03 单一工作流定义加载器

新增 `docs/.workflow/workflow_definition.json`，包含：

- feature/bug 阶段 canonical/display/order/alias。
- 子 Agent 名称、允许阶段和 prerequisites。
- auto gate 的 from/to/checklist/allowed/blocked。
- context packet 的 must_read/on_demand/outputs。

新增 `docs/.workflow/scripts/workflow_def.py` 负责加载、默认值和类型校验。第一阶段保持等价：脚本中可保留同名常量作为加载后的缓存，但不得继续维护多份手写表。

覆盖: R04

### D04 Gate-owned validator 执行

将 `validators.py` 的各校验函数拆成可 import 的无副作用函数，返回结构化结果；CLI 只打印结果，不再推进 state。`stage_gates.py auto <gate>` 根据 gate 定义调用绑定 validator，通过后才转移。

覆盖: R05

### D05 声明式前置条件与状态元数据

在 gate 定义中声明 prereqs，统一求值：

- `file_glob`
- `keyword_in_file`
- `checklist_true`
- `state_field`
- `artifact_exists`
- `validator_passed`

新增或规范化 state 字段：

- `workflow_runtime.docs_only`
- `workflow_runtime.worktree_created`
- `workflow_runtime.build_config_overridden`
- `workflow_runtime.validation_results`

旧 state 缺字段时按默认值读取；迁移期允许 Markdown 证据作为 fallback，但写入动作必须更新 state。

覆盖: R06, R07

### D06 上下文包模板化

把 feature 和 bug 上下文包正文迁移到 `docs/.workflow/templates/上下文包.md.tpl` 和 `docs/.workflow/templates/bug上下文包.md.tpl`。`context_packets.py` 只负责组装数据和渲染模板。

当 must-read 未命中时，结合 `in_progress_step.expected_outputs` 与 `done_definition` 生成“待生成清单”。

覆盖: R08

### D07 公共工具模块和配置 schema

新增 `workflow_common.py`：

- JSON 读取与快照 fallback。
- state 保存与 `last_updated` 更新。
- step/progress/subagent JSON 校验。
- 子 Agent blocking 状态过滤。

扩展 `workflow_config.py`：

- build/workflow 配置统一加载。
- 类型错误明确报错。
- 缺失键使用默认并返回 warning 信息。

覆盖: R09

### D08 低摩擦命令

在不削弱强校验前提下改造 `stage_gates.py`：

- `subagent-done` 结果 JSON 不含 `dispatch_id` 时，若当前步骤仅一条未闭合派遣则自动匹配。
- 多条未闭合派遣必须显式提供 `dispatch_id`。
- `step-done` 可省略步骤名，默认当前 `in_progress_step.step`。
- 文档给出最短调用示例。

覆盖: R10

### D09 语言无关文档叙事

更新 `README.md`、`INSTALL.md`、`使用指南.md`、`工作流规范.md §11`：

- 说明 workflow-kit 语言无关。
- Java/Maven/Jar 只是默认 build profile。
- 给出前端 `dist/**/*` 和 Go `bin/*` 配置示例。

覆盖: R11

## 风险控制

- 每个 T 单独实现、验证和记录，避免一次性重写 2500+ 行 `stage_gates.py`。
- `validators.py` 去副作用前，所有自动转移测试必须先锁定。
- 单一事实源第一阶段只做等价迁移，不支持项目随意增删 S1-S10 阶段。
- 安装脚本必须使用标准库，不能引入第三方安装依赖。
- 所有写 state 的命令串行执行，避免快照清理竞争。
