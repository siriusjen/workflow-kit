# 工作流自动化优化 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 workflow-kit 的安装、门禁、配置和文档自动化能力按可验证任务逐步落地。

**Architecture:** 先修安装入口和测试安全网，再做状态机内部重构。所有脚本实现都在代码 worktree 中完成；主项目目录只维护 feature 文档和状态机记录。

**Tech Stack:** Python 标准库、unittest、Markdown/JSON 工作流文档、git worktree。

---

## File Structure

- Create `docs/.workflow/scripts/install_entry.py`: 入口受管块注入 CLI。
- Create `docs/.workflow/templates/entry/AGENTS.md.tpl`: Codex 入口受管块模板。
- Create `docs/.workflow/templates/entry/CLAUDE.md.tpl`: Claude Code 入口受管块模板。
- Create `docs/.workflow/scripts/tests/test_install_entry.py`: 安装脚本测试。
- Modify `AGENTS.md`, `CLAUDE.md`: 加 `WORKFLOW-KIT` 受管块边界。
- Modify `INSTALL.md`, `使用指南.md`, `docs/.workflow/工作流规范.md`: 安装说明和语言无关叙事。
- Modify `docs/.workflow/scripts/tests/test_workflow_gates.py`: 门禁回归测试扩展。
- Later tasks create `workflow_definition.json`, `workflow_def.py`, `workflow_common.py`, context templates, and refactor existing scripts.
