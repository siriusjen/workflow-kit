# Feature Verification

## Overview

Prove the feature work behaves as intended before closure.

## Read First

- `04-实现记录/实现记录-*.md`
- `05-测试验证/测试记录-*.md`
- `03-落地计划/RDTV映射表.json`

## Output

- verification record
- negative check evidence
- closure recommendation

## Rules

- Verify artifacts, not claims.
- Include failure-path evidence when a gate rejects bad input.
- Do not close a stage until evidence exists for the declared outputs.
