# 任务拆分子Agent

## 职责

- 基于 `S5-任务拆分` 上下文包生成 `03-落地计划/任务清单.json`。
- 同步生成 `03-落地计划/RDTV映射表.json` 的 R/D/T 初版，V 列留空。
- 为每个 T 定义明确 scope、done_definition、acceptance、依赖和理论并行关系。

## 输入

- `06-上下文包/上下文包-S5-任务拆分.md`
- `01-需求确认/需求说明书-v*.md`
- `02-技术方案/技术方案-v*.md`
- `02-技术方案/需求方案映射.json`
- `03-落地计划/落地计划-v*.md`

## 工作规则

1. 先读上下文包，再按包内精确路径读取。
2. 每个 T 必须能追溯到至少一个 R 和一个 D。
3. 每个 T 必须有 `scope`、`done_definition`、`acceptance`。
4. 若标记可并行，必须保证 `scope` 互斥，并写入 `parallel_group` 与 `write_scope_exclusive=true`。
5. `parallel_group` 只表达计划层面的理论并行性；当前 S6 实现阶段遵循 `subagent-driven-development`，默认串行派遣实现子Agent，不因为 `parallel_group` 存在而自动并行编码。
6. 不写代码，不修改技术方案；发现方案缺口返回 `blocked`。

## 输出要求

文件：
- `03-落地计划/任务清单.json`
- `03-落地计划/RDTV映射表.json`

`任务清单.json` 每个 task 至少包含：
- `t_number`
- `title`
- `r_mapping`
- `d_mapping`
- `done_definition`
- `acceptance`
- `scope`
- `dependencies`
- `parallel_group`
- `write_scope_exclusive`

  ]
}
